Структура K8s
```bash
The-Box/
├── k8s/
│   ├── Chart.yaml -- Манифест для K8s что бы он знал, перед ним офицальный пакет
│   ├── values.yaml -- Единный центр управления всеми сервисами 
│   └── templates/
│       ├── configmap.yaml -- Файл переменных окружения кластера K8s
│       ├── deployment.yaml -- Умный шаблон деплоя
│       └── service.yaml -- Для проброса внутренних сетевых портов \(ClusterIP\)\ внутри кластера, для общения сервисов между собой и до них мог достучаться будущий API Gateway \(Kong\)\

```

Развернут полноценный, гибкий **Umbrella Helm-чарт**[values.yaml], циклы, правильное разделение на values и шаблоны, лимиты по CPU и памяти (limits/requests), а также подтягивание конфигов через envFrom - прописывание через переменные из [configmap.yaml] => [deployment.yaml].
Надо сделать проверку синтаксиса Helm прямо в терминале хоста (для этого k8s не нужен, нужен просто установленный Helm на Макбуке).

```bash
helm template ./k8s

```
